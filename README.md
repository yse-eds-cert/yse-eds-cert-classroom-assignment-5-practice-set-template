## How to Open the Assignment Notebook

1. You have already **accepted the assignment from Canvas** via the invitation link. When you did that, GitHub Classroom created your personal repo (e.g., `yse-eds-cert/assignment-5-<yourUserNameHere>`).
2. After you accepted the assignment, you then **went to your personal repo** on GitHub (e.g., `https://github.com/yse-eds-cert/assignment-5-<yourUserNameHere>`, *Note:* `<yourUserNameHere>` is *your* GitHub username).
3. **Open the assignment notebook in Google Colab.** While orienting yourself, you can see this GitHub repo contains a file named `C2M5_practice-set.ipynb`. Next, we need to launch that notebook in Google Colab:

   - **Option A (Recommended)** — Open Google Colab directly:
     1. Open https://colab.research.google.com/
     2. Click the **GitHub** tab in the pop-up
     3. For the repository, choose `yse-eds-cert/assignment-5-<yourUserNameHere>`, replacing `<yourUserNameHere>` with *your* GitHub username
     4. Click `C2M5_practice-set.ipynb`

   - **Option B** — Edit the following link with your GitHub username, replacing `<yourUserNameHere>` with *your* GitHub username, and follow that edited link:
     `https://colab.research.google.com/github/yse-eds-cert/assignment-5-<yourUserNameHere>/blob/main/C2M5_practice-set.ipynb`

4. In Colab, go to **File → Save a copy in Drive** — this creates an autosaving working copy in your Google Drive. Do all your work there, so that you always then have a copy backed up in your personal Drive.
5. Follow the instructions in the notebook to **Complete the Practice Set**. Be sure to follow the instructions at the very beginning of the Notebook that tell you how to make sure the Notebook expects the correct file paths.
6. When finished, submit by going **File → Save a copy in GitHub** → select your personal assignment repo (`yse-eds-cert/assignment-5-<yourUserNameHere>`) and "push" your final work to your GitHub repo.

> **Why save to Drive first?** Colab autosaves to Google Drive but does **not** autosave to GitHub. Saving to Drive protects your work-in-progress. Saving to GitHub is the manual submission step.
